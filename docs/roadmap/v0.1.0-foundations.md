- **Select** = form control for single selection
- **Menu** = navigation menus (menu bar, sidebar nav)

## Why This Milestone

### Validates Core Patterns

v0.1.0 establishes how askr-ui wraps and uses askr/foundations primitives:

1. **Use existing foundations** — Slot, Portal, Presence, pressable(), dismissable(), etc.
2. **Wrap foundations cleanly** — FocusScope, DismissableLayer wrap foundation logic
3. **Controlled + uncontrolled** — use controllableState() from foundations
4. **Accessibility baked in** — leverage foundation ARIA helpers
5. **Deterministic updates** — build on askr's scheduler

### Enough Surface Area

These 15 UI components provide enough variety to validate:
- Simple components (Button, Input, Label)
- Compound components (Dialog, Select, RadioGroup)
- Overlay positioning (Tooltip, Popover)
- Focus management (FocusScope, DismissableLayer)
- Form patterns (Field wrapper, validation states)

### Enables Real Apps

With just v0.1.0, developers can build:
- Complete authentication flows
- Settings and preferences UIs
- Contact and feedback forms
- Basic task management apps
- Simple CRUD interfaces

This proves the library is **useful from day one**, not just a collection of primitives.

---

## Not Included

v0.1.0 **intentionally excludes**:

- ❌ Advanced form controls (DatePicker, Combobox, NumberField) — wait for basic patterns to stabilize
- ❌ Navigation components (Tabs, Accordion, Breadcrumb) — focus on forms/overlays first
- ❌ Data display (Table, TreeView) — form entry before data display
- ❌ Layout helpers (ResizablePanel, ScrollArea) — content before chrome
- ❌ Feedback components (Toast, Progress, Skeleton) — core interactions first

These come in later milestones after foundational patterns are proven.

## Implementation Order

Recommended build sequence within v0.1.0:

### Phase 1: UI Utility Components
1. **VisuallyHidden** — screen-reader-only wrapper
2. **Separator** — semantic divider component
3. **FocusRing** — visible focus indicators
4. **FocusScope** — focus containment wrapper
5. **DismissableLayer** — dismissable interaction wrapper

### Phase 2: Basic Inputs
6. **Label** — required for all form fields
7. **Input** — most common form control
8. **Textarea** — extends Input patterns
9. **Field** — wrapper with helper/error text

### Phase 3: Selection Controls
10. **Checkbox** — basic boolean selection
11. **RadioGroup** — single selection pattern
12. **Switch** — iOS-style toggle

### Phase 4: Overlays (use askr/foundations primitives)
13. **Dialog** — general modal pattern
14. **AlertDialog** — confirmation dialogs
15. **Popover** — positioned content
16. **Tooltip** — hover labels

### Phase 5: Selection / Actions
17. **Select** — dropdown selection
18. **Menu** — navigation menus
19. **DropdownMenu** — action menus

## Success Criteria

v0.1.0 is complete when:

✅ All 15 UI components shipped and documented  
✅ All components have behavior, a11y, and determinism tests  
✅ askr/foundations primitives (Slot, Portal, Presence, etc.) used consistently across components  
✅ Prop merging and event composition working consistently  
✅ Controlled/uncontrolled state patterns validated  
✅ Overlay positioning and focus management working  
✅ Developers can build complete authentication and settings UIs  
✅ No show-stopping API issues discovered

Once v0.1.0 is stable, the patterns scale to v0.2.0+ components.
