# v0.3.0 — Data Entry and Rich Forms

**Status:** Planned  
**Target:** Serious CRUD apps, internal tools, workflows with rich validation

---

## What You Can Build

Building on v0.1.0's basic forms and v0.2.0's navigation, v0.3.0 adds:

- ✅ **Booking and scheduling interfaces** — date/time pickers with calendars
- ✅ **Search interfaces with autocomplete** — fast keyboard-driven selection
- ✅ **Command palettes for power users** — Cmd+K style quick actions
- ✅ **Complex multi-step workflows** — structured forms with field groups
- ✅ **E-commerce checkout flows** — address entry, date selection, payment forms
- ✅ **CRM and sales tools** — contact forms with searchable dropdowns
- ✅ **Data entry applications** — numeric fields with validation and formatting
- ✅ **Authentication with OTP** — PIN input for two-factor codes
- ✅ **Rich hover cards** — show user profiles or previews on hover
- ✅ **Context menus** — right-click for context-specific actions
- ✅ **Slide-out panels** — Sheet/Drawer for side panels and mobile navigation

## Components (14)

### Advanced Form Controls (9)

Rich input components for complex data entry:

| Component | Purpose |
|-----------|---------|
| **NumberField** | Numeric input with increment/decrement buttons and formatting |
| **PinInput** | Multi-digit PIN or OTP code entry |
| **Combobox** | Searchable/filterable selection with keyboard navigation |
| **Autocomplete** | Text input with inline suggestions and completion |
| **CommandPalette** | Cmd+K style quick action and navigation menu |
| **DatePicker** | Date selection with calendar popup |
| **Calendar** | Standalone calendar for date selection |
| **TimeField** | Time input with hours/minutes/seconds |
| **RangeCalendar** | Date range selection (start and end dates) |

**Differentiation:**
- **Select (v0.1.0)** = simple dropdown, single selection
- **Combobox (v0.3.0)** = searchable selection, filters as you type
- **Autocomplete (v0.3.0)** = free-form text input with suggestions

### Form Structure (4)

Organize complex forms:

| Component | Purpose |
|-----------|---------|
| **Fieldset** | Group related form fields with legend |
| **Form** | Form wrapper with submission handling and validation orchestration |
| **InputGroup** | Attach addons (icons, text, buttons) to inputs |
| **SearchField** | Input with search icon and clear button |

### Additional Overlays / Pickers (3)

Richer overlay patterns:

| Component | Purpose |
|-----------|---------|
| **HoverCard** | Rich interactive content on hover (vs Tooltip for simple text) |
| **ContextMenu** | Right-click or long-press contextual menus |
| **Sheet** | Slide-in panel from edge of screen (also called Drawer) |

**Differentiation:**
- **Tooltip (v0.1.0)** = simple text label on hover
- **HoverCard (v0.3.0)** = rich content (user cards, previews) on hover with delay
- **Popover (v0.1.0)** = click-to-open positioned content
- **Menu (v0.1.0)** = explicit navigation menus
- **ContextMenu (v0.3.0)** = right-click contextual action menus

## Why This Milestone

### Completes the Data Entry Story

v0.1.0 provided basic inputs (text, checkbox, radio, select). v0.3.0 adds the missing pieces for serious business applications:

| Use Case | Components |
|----------|------------|
| **Booking system** | DatePicker, RangeCalendar, TimeField |
| **Search interface** | Combobox, Autocomplete, SearchField |
| **Power user tools** | CommandPalette with keyboard shortcuts |
| **Financial apps** | NumberField with currency formatting |
| **Authentication** | PinInput for 2FA codes |
| **Mobile-optimized UIs** | Sheet for slide-out navigation |

### Learns from v0.1.0 Selection Patterns

**Why wait for v0.3.0?**

- **Combobox** builds on Select (v0.1.0) patterns — needs simple dropdown validated first
- **DatePicker** uses Popover (v0.1.0) positioning — overlay patterns must be stable
- **CommandPalette** needs Menu (v0.1.0) and Dialog (v0.1.0) patterns proven
- **NumberField** extends Input (v0.1.0) — wait for text input to be solid

v0.3.0 components are more complex and benefit from lessons learned in v0.1.0.

### Enables Complex Workflows

With v0.1.0 + v0.2.0 + v0.3.0, you can build complete business applications:

- **CRM tools** — search contacts with Combobox, schedule meetings with DatePicker
- **Booking platforms** — date range selection, time pickers, multi-step forms
- **Admin dashboards** — command palette for quick actions, context menus for row operations
- **E-commerce** — address forms with autocomplete, date selection for deliveries

## Builds on v0.1.0 + v0.2.0

v0.3.0 components leverage earlier foundations:

- **Combobox** → uses **Popover** (v0.1.0) + **Input** (v0.1.0) + **Menu** patterns
- **DatePicker** → uses **Popover** (v0.1.0) + **Button** (v0.1.0) trigger
- **Calendar** → uses **FocusScope** (v0.1.0) for keyboard navigation
- **CommandPalette** → uses **Dialog** (v0.1.0) + **Menu** (v0.1.0) patterns
- **NumberField** → extends **Input** (v0.1.0) with stepper buttons
- **Sheet** → uses **Portal** (v0.1.0), **FocusScope** (v0.1.0), **DismissableLayer** (v0.1.0)
- **HoverCard** → uses **Popover** (v0.1.0) positioning with hover trigger
- **ContextMenu** → uses **Menu** (v0.1.0) patterns with different trigger

No new interaction primitives required — all build on proven v0.1.0 foundations.

---

## Not Included

v0.3.0 **intentionally excludes**:

- ❌ Data tables — display patterns come after input patterns
- ❌ Tree views — hierarchical display should wait for table patterns
- ❌ Data grids — very complex, needs simpler collections validated first
- ❌ Rich text editors — wait for all form patterns to stabilize
- ❌ File uploads — adds async complexity, wait for core forms

These come in v0.4.0+ after form patterns are fully proven.

## Success Criteria

v0.3.0 is complete when:

✅ All 14 components shipped and documented  
✅ DatePicker has calendar keyboard navigation (arrow keys, home/end, page up/down)  
✅ Combobox handles large option lists performantly (1000+ items)  
✅ CommandPalette works with keyboard shortcuts and fuzzy search  
✅ NumberField handles formatting (currency, percentages, thousands separators)  
✅ PinInput auto-focuses next field and handles paste  
✅ RangeCalendar supports start/end selection with proper states  
✅ HoverCard has appropriate hover delay (not instant)  
✅ ContextMenu works via right-click, long-press, and keyboard (Shift+F10)  
✅ Sheet has smooth slide-in animations and proper focus restoration  
✅ Developers can build complete booking and CRM applications
