# v0.5.0 — Advanced Patterns and Polish

**Status:** Planned  
**Target:** Enterprise interfaces, performance-critical UIs, advanced keyboard navigation

## What You Can Build

Building on the complete foundation of v0.1.0–v0.4.0, v0.5.0 adds:

- ✅ **Rich text editors** — floating toolbars with formatting controls
- ✅ **Data grids with thousands of rows** — virtualized, sortable, filterable enterprise grids
- ✅ **Onboarding flows** — product tours with step-by-step guides
- ✅ **Command-driven interfaces** — keyboard-first navigation with command bars
- ✅ **Complex application shells** — complete layouts with sidebars, panels, and toolbars
- ✅ **Performance-optimized lists** — handle 10,000+ items with smooth scrolling
- ✅ **Async workflow UIs** — loading overlays and pending state management
- ✅ **Pinterest-style layouts** — masonry grids for images and cards
- ✅ **Desktop-class web applications** — roving focus, advanced keyboard shortcuts
- ✅ **Inline editing interfaces** — click-to-edit tables and lists

## Components (15+)

### Advanced Interaction (4)

Sophisticated keyboard and interaction patterns:

| Component            | Purpose                                                                    |
| -------------------- | -------------------------------------------------------------------------- |
| **Toolbar**          | Horizontal toolbar with button groups and segmented controls               |
| **RovingFocusGroup** | Single-tab-stop navigation with arrow keys (primitive for custom patterns) |
| **Stepper**          | Multi-step workflow indicator with progress                                |
| **Tour**             | Product tour/coachmark primitives for guided onboarding                    |

### Advanced Overlays / Windows (2)

Additional overlay patterns:

| Component           | Purpose                                                                    |
| ------------------- | -------------------------------------------------------------------------- |
| **Tray**            | Bottom sheet or slide-up panel (mobile-first)                              |
| **FloatingToolbar** | Context-sensitive toolbar that floats near selection (text editor pattern) |

**Differentiation:**

- **Sheet (v0.3.0)** = slide-in from edge (left/right)
- **Tray (v0.5.0)** = slide-up from bottom (mobile pattern)
- **Toolbar (v0.5.0)** = static horizontal toolbar
- **FloatingToolbar (v0.5.0)** = positioned toolbar that follows selection

### Advanced Collections (3)

Performance-optimized display patterns:

| Component       | Purpose                                                               |
| --------------- | --------------------------------------------------------------------- |
| **DataGrid**    | Enterprise-grade data grid with inline editing, grouping, aggregation |
| **VirtualList** | Virtualized list rendering for 10,000+ items                          |
| **Masonry**     | Pinterest-style layout with variable-height items                     |

**Why wait until v0.5.0?**

- **DataGrid** is extremely complex — needs Table (v0.4.0) patterns proven first
- **VirtualList** is performance optimization — needs simple Listbox (v0.4.0) validated
- **Masonry** is layout complexity — build after ScrollArea (v0.4.0) patterns work

### Feedback / Async UX (3)

Handle loading and async workflows:

| Component           | Purpose                                                             |
| ------------------- | ------------------------------------------------------------------- |
| **LoadingOverlay**  | Full-screen or scoped loading state overlay                         |
| **PendingBoundary** | Async state management primitive (loading/error/success boundaries) |
| **InlineEditable**  | Click-to-edit fields for tables and lists                           |

**Differentiation:**

- **Progress (v0.2.0)** = determinate progress bar
- **Spinner (v0.2.0)** = indeterminate circular loading
- **Skeleton (v0.2.0)** = content placeholder during load
- **LoadingOverlay (v0.5.0)** = full overlay for async operations
- **PendingBoundary (v0.5.0)** = state management wrapper for async UI

### App Primitives (3+)

Complete application shell components:

| Component      | Purpose                                                               |
| -------------- | --------------------------------------------------------------------- |
| **CommandBar** | Persistent command bar with shortcuts and search (like VSCode)        |
| **Sidebar**    | Collapsible sidebar with navigation items                             |
| **AppShell**   | Complete application layout primitive (header, sidebar, main, footer) |

## Why This Milestone

### Maturity Release

v0.5.0 is **not** about adding essential components — it's about **polish and advanced patterns** for demanding applications.

**Characteristics:**

- Components are "nice to have" rather than "must have"
- Patterns are proven in v0.1.0–v0.4.0 before building advanced versions
- Performance optimizations (virtualization) come after simple patterns work
- Enterprise features (DataGrid) ship after core is battle-tested

### Completes the Library

After v0.5.0, askr-ui has:

- ✅ Complete form controls (v0.1.0 + v0.3.0)
- ✅ All overlay patterns (v0.1.0 + v0.3.0 + v0.5.0)
- ✅ Navigation and disclosure (v0.2.0)
- ✅ Data display and collections (v0.4.0 + v0.5.0)
- ✅ Layout and application shell (v0.4.0 + v0.5.0)
- ✅ Performance optimization (v0.5.0)
- ✅ Advanced keyboard UX (v0.5.0)

No major gaps remain for building production applications.

### Avoids Early Mistakes

By shipping v0.5.0 **last**, we avoid:

- ❌ Locking in bad DataGrid APIs before Table patterns mature
- ❌ Building virtualization before simple collection patterns work
- ❌ Shipping complex app shells before component composition is proven
- ❌ Creating tour/onboarding patterns before overlay patterns stabilize

v0.5.0 benefits from **four previous milestones** of lessons learned.

## Builds on v0.1.0–v0.4.0

v0.5.0 is possible because earlier milestones provided:

- **DataGrid** → builds on **Table** (v0.4.0), **Input** (v0.1.0), **Combobox** (v0.3.0)
- **VirtualList** → optimizes **Listbox** (v0.4.0) proven pattern
- **FloatingToolbar** → uses **Popover** (v0.1.0) positioning, **Toolbar** (v0.5.0) content
- **Tour** → uses **Dialog** (v0.1.0) or **Popover** (v0.1.0) for steps
- **CommandBar** → combines **CommandPalette** (v0.3.0) + **Menu** (v0.1.0)
- **LoadingOverlay** → uses **Portal** (v0.1.0), **FocusScope** (v0.1.0)
- **AppShell** → uses **Sidebar** (v0.5.0), **NavigationMenu** (v0.2.0), **Card** (v0.4.0)
- **InlineEditable** → uses **Input** (v0.1.0), **NumberField** (v0.3.0), focus management

Everything builds on proven primitives — no new core patterns introduced.

## Performance Targets

v0.5.0 components must meet high performance standards:

| Component            | Target                                            |
| -------------------- | ------------------------------------------------- |
| **VirtualList**      | 10,000+ items, 60fps scroll, <100ms first paint   |
| **DataGrid**         | 100,000 cells, <16ms cell render, instant sorting |
| **Masonry**          | 1,000+ items, smooth resize, efficient layout     |
| **InlineEditable**   | <50ms edit mode transition                        |
| **RovingFocusGroup** | <1ms focus change, handles 1000+ items            |

These are achieved through:

- Virtualization (only render visible items)
- Memoization (cache expensive computations)
- Incremental rendering (Askr's scheduler)
- Efficient DOM updates (minimal thrashing)
