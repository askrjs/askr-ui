## Components (15 net-new)

### Disclosure / Reveal (3)

Expandable and collapsible content patterns:

| Component       | Purpose                                                                      |
| --------------- | ---------------------------------------------------------------------------- |
| **Accordion**   | Multiple collapsible panels with single-select behavior (one open at a time) |
| **Collapsible** | Single expandable/collapsible region                                         |
| **Tabs**        | Tabbed interface with keyboard navigation (ARIA tabs pattern)                |

**Differentiation:**

- **Accordion** = multiple panels, typically only one open
- **Collapsible** = single region, can be used standalone or in custom accordions
- **Tabs** = organize content into named tabs with panel switching

### Navigation (4)

Page-level and application navigation:

| Component          | Purpose                                            |
| ------------------ | -------------------------------------------------- |
| **Breadcrumb**     | Hierarchical navigation trail                      |
| **Pagination**     | Navigate through paginated data                    |
| **NavigationMenu** | Multi-level dropdown navigation (for site headers) |
| **Menubar**        | Horizontal menu bar (File, Edit, View pattern)     |

### Utility / Feedback (6)

Status indicators and loading states:

| Component          | Purpose                                                                 |
| ------------------ | ----------------------------------------------------------------------- |
| **Badge**          | Notification counts and indicators                                      |
| **Avatar**         | User profile pictures with fallback initials                            |
| **Skeleton**       | Loading placeholder that mimics content shape                           |
| **ProgressCircle** | Circular/radial progress indicator (includes Spinner for indeterminate) |
| **Progress**       | Linear progress bar                                                     |
| **Toast**          | Temporary notification messages                                         |

**Differentiation:**

- **ProgressCircle** = circular/radial indicator, can be determinate or indeterminate (spinner)
- **Progress** = horizontal linear progress bar

### Additional Input Controls (2 new + Toggle reference)

Extended form inputs:

| Component       | Purpose                                                  |
| --------------- | -------------------------------------------------------- |
| **Toggle**      | Button-like pressed state (already shipped ✅ in v0.0.1) |
| **ToggleGroup** | Multiple exclusive or independent toggles                |
| **Slider**      | Numeric range input with thumb control                   |

**Note:** Toggle already exists (aria-pressed button). Switch (v0.1.0) is different (iOS-style role="switch").

## Why This Milestone

### Completes Application Shells

v0.1.0 gave you forms and overlays. v0.2.0 adds everything needed for complete application layouts:

- Navigation between pages (Breadcrumb, NavigationMenu, Menubar)
- Content organization (Tabs, Accordion, Collapsible)
- Pagination for long lists
- Loading and status feedback

### Enables Common UX Patterns

With v0.2.0, you can build:

| Pattern                | Components Used                                                      |
| ---------------------- | -------------------------------------------------------------------- |
| **Dashboard layout**   | NavigationMenu, Breadcrumb, Badge, Avatar, Toast                     |
| **Settings panel**     | Tabs, Toggle, ToggleGroup, Slider, Collapsible                       |
| **Documentation site** | Accordion for FAQs, Tabs for API sections, Breadcrumb for navigation |
| **Admin table UI**     | Pagination, Badge for status, Skeleton for loading, Progress         |
| **Feature tours**      | Toast for announcements, Badge for new feature indicators            |

### Validates Disclosure Patterns

The disclosure components (Accordion, Collapsible, Tabs) test:

- Controlled expand/collapse state
- Keyboard navigation between panels/tabs
- ARIA relationships (aria-controls, aria-expanded)
- Single-select vs multi-select behavior

These patterns will inform v0.3.0+ compound components.

## Builds on v0.1.0

v0.2.0 components leverage v0.1.0 foundations:

- **Tabs** → uses **FocusScope** and **Pressable** from v0.1.0
- **NavigationMenu** → uses **Popover** positioning and **Menu** patterns
- **Accordion** → uses **Collapsible** as building block
- **Toast** → uses **Portal** to escape DOM hierarchy
- **Skeleton** → uses **VisuallyHidden** for screen reader text
- **ToggleGroup** → uses **Toggle** as base component

No v0.2.0 component introduces patterns that contradict v0.1.0.

---

## Not Included

v0.2.0 **intentionally excludes**:

- ❌ Advanced search (Combobox, Autocomplete, CommandPalette) — wait for selection patterns to mature in v0.1.0
- ❌ Date/time pickers — complex overlays should wait for simpler overlays to prove stable
- ❌ Data tables — display patterns come after navigation patterns
- ❌ Tree views — hierarchical navigation should build on proven menu/accordion patterns
- ❌ Resizable panels — layout helpers come after content components

These come in v0.3.0+ after navigation patterns are validated.
