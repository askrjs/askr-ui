# v0.4.0 — Collections and Data Display

**Status:** Planned  
**Target:** Complex admin UIs, explorer-style interfaces, information-dense applications

---

## What You Can Build

Building on v0.1.0–v0.3.0's input and navigation capabilities, v0.4.0 adds:

- ✅ **Data tables with sorting and selection** — display datasets with interactive columns
- ✅ **File explorers and directory trees** — hierarchical navigation with expand/collapse
- ✅ **Status dashboards with metrics** — display KPIs and statistics
- ✅ **Document viewers with outlines** — navigable table of contents
- ✅ **Split-pane editing interfaces** — resizable panels for code editors, email clients
- ✅ **Analytics dashboards** — cards with grouped metrics and trends
- ✅ **Admin list views** — listboxes with selection and keyboard navigation
- ✅ **Empty states** — graceful handling of no-data scenarios
- ✅ **Keyboard shortcut documentation** — Kbd component for displaying keys
- ✅ **Alert banners** — page-level status messages
- ✅ **Scrollable regions** — custom scroll areas with styling control

## Components (15)

### Data Display (7)

Components for displaying structured information:

| Component      | Purpose                                                 |
| -------------- | ------------------------------------------------------- |
| **Table**      | Tabular data with rows, columns, headers, sorting       |
| **DataList**   | Key-value pairs (definition lists)                      |
| **Listbox**    | Selectable list with keyboard navigation (ARIA listbox) |
| **EmptyState** | Placeholder for empty data views                        |
| **Kbd**        | Keyboard shortcut display (styled `<kbd>` elements)     |
| **Stat**       | Numeric metric display with label and trend indicator   |
| **Card**       | Content container with optional header/footer           |

### Tree / Hierarchical Interaction (2)

Navigate hierarchical data:

| Component      | Purpose                                           |
| -------------- | ------------------------------------------------- |
| **TreeView**   | Hierarchical tree navigation with expand/collapse |
| **NestedMenu** | Multi-level hierarchical menus                    |

**Differentiation:**

- **TreeView** = file explorer pattern, arbitrary depth, selection states
- **NestedMenu** = navigation menus with submenus

### Feedback / Status (4)

Page-level and inline status indicators:

| Component       | Purpose                                                              |
| --------------- | -------------------------------------------------------------------- |
| **InlineAlert** | Inline validation/error messages in forms (contextual)               |
| **Callout**     | Emphasized content blocks (tips, warnings in documentation)          |
| **Banner**      | Page-level alert or announcement bar                                 |
| **Status**      | Status labels and categorical markers (success, warning, error tags) |

**Differentiation:**

- **Badge (v0.2.0)** = notification counts, small indicators
- **Status (v0.4.0)** = status labels with semantic colors (success, warning, error)
- **InlineAlert (v0.4.0)** = inline form validation messages
- **Callout (v0.4.0)** = emphasized blocks in documentation
- **Banner (v0.4.0)** = page-level alerts and announcements
- **Toast (v0.2.0)** = temporary auto-dismissing notifications

### Layout Helpers (4)

Control layout and spacing:

| Component          | Purpose                                           |
| ------------------ | ------------------------------------------------- |
| **AspectRatio**    | Maintain aspect ratio for media (16:9, 4:3, etc.) |
| **ScrollArea**     | Custom scrollable region with styled scrollbars   |
| **ResizablePanel** | User-resizable panels with drag handles           |
| **Splitter**       | Divider between resizable panels                  |

## Why This Milestone

### Completes the Display Story

v0.1.0–v0.3.0 focused on **data entry** (forms, inputs, pickers). v0.4.0 adds **data display** (tables, trees, lists):

| Use Case               | Components                                       |
| ---------------------- | ------------------------------------------------ |
| **Admin dashboard**    | Table, Stat, Card, Banner, Status                |
| **File explorer**      | TreeView, NestedMenu, Kbd for shortcuts          |
| **Code editor**        | ResizablePanel, Splitter, ScrollArea             |
| **Documentation site** | Callout, Card, TreeView for TOC                  |
| **Analytics UI**       | Table with sorting, Stat for KPIs, EmptyState    |
| **Email client**       | Listbox for messages, ResizablePanel for preview |

### Balances Input with Output

After three milestones focused on entering and selecting data, v0.4.0 shifts to **browsing and displaying** data:

- v0.1.0–v0.3.0: "How do I enter data?"
- v0.4.0: "How do I display it?"

This balance makes askr-ui suitable for applications where **reading data** is as important as **writing it**.

### Enables Information-Dense UIs

v0.4.0 components shine in interfaces with high information density:

- **Admin panels** — tables with hundreds of rows, status indicators, quick filters
- **Developer tools** — resizable code panels, tree navigation, keyboard shortcuts
- **Analytics dashboards** — cards full of metrics, charts, and trends
- **Document editors** — split views, hierarchical outlines, scrollable canvases

---

## Builds on v0.1.0–v0.3.0

v0.4.0 components leverage all previous foundations:

- **Table** → uses **Checkbox** (v0.1.0) for row selection, **Menu** (v0.1.0) for column actions
- **TreeView** → uses **Collapsible** (v0.2.0) for expand/collapse, **FocusScope** (v0.1.0) for keyboard nav
- **Listbox** → uses **FocusScope** (v0.1.0) and roving focus from askr/foundations
- **ResizablePanel** → uses **Separator** (v0.1.0) as drag handle
- **Card** → uses **Slot** (v0.1.0) for header/footer composition
- **Banner** → uses **Portal** (v0.1.0) to render at top of viewport
- **Stat** → uses **Card** (v0.4.0) as optional container

No new primitives required — builds entirely on proven patterns.

---

## Not Included

v0.4.0 **intentionally excludes**:

- ❌ DataGrid — very complex, needs Table patterns validated first (comes in v0.5.0)
- ❌ VirtualList — performance optimization should wait for simple lists to prove stable
- ❌ Rich text editors — wait for all text input patterns to mature
- ❌ Charts and graphs — visualization is outside askr-ui's scope (headless library)
- ❌ Drag-and-drop reordering — complex interaction pattern, wait for stable foundations

These advanced patterns come in v0.5.0 after simpler collections are proven.

## Success Criteria

v0.4.0 is complete when:

✅ All 15 components shipped and documented  
✅ Table supports sorting, selection, and keyboard navigation  
✅ TreeView handles deeply nested hierarchies (100+ nodes)  
✅ Listbox has proper keyboard navigation (arrows, home/end, type-ahead)  
✅ ResizablePanel supports both horizontal and vertical splits  
✅ ScrollArea has custom scrollbar styling and proper touch support  
✅ Status indicators have semantic colors and proper ARIA  
✅ EmptyState handles various empty scenarios gracefully  
✅ Card supports flexible header/footer composition via Slot  
✅ Banner can be dismissed and persists preference  
✅ Developers can build complete admin dashboards and file explorers
